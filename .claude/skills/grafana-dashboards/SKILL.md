## Golden Rules (Never Break These)

1. **Generate JSON with Python** — use builder functions, never write raw JSON.
2. **Grid is exactly 24 columns.** Every panel: `x + w <= 24`. Validate before output.
3. **Add `y += 1` after every row panel.** Rows consume 1 unit of vertical space.
4. **`schemaVersion: 42`**, `pluginVersion: "12.4.0"` for Grafana 11/12.
5. **Datasource ref on every target AND every variable:**
   `{"type": "prometheus", "uid": "${DS_PROMETHEUS}"}` — use `copy.deepcopy(DS)`.
6. **Never join inside a range function.**
   `rate(metric * on(id) info[5m])` = INVALID.
   `rate(metric[5m]) * on(id) group_left(name) info` = VALID.
7. **Table targets need `format: "table"` + `instant: True`.**
8. **Collapsed row child panels live inside `row["panels"]`, NOT in top-level panels.**
9. **`allValue: ".*"` on every multi-select variable** or "All" sends literal text to Prometheus.
10. **`definition` field must exactly match `query.query`** or variable breaks on reload.

```python
import json, copy

DS = {"type": "prometheus", "uid": "${DS_PROMETHEUS}"}

def tgt(expr, legend="", ref="A", fmt="time_series", instant=False):
    t = {"datasource": copy.deepcopy(DS), "expr": expr,
         "legendFormat": legend, "refId": ref}
    if fmt != "time_series": t["format"] = fmt
    if instant: t["instant"] = True
    return t

def _tf(id, options=None):
    return {"id": id, "options": options or {}}

panels = []
y = 0

# --- build panels (see references/panel-types.md for builders) ---

# --- validate ---
for p in panels:
    gp = p.get("gridPos", {})
    assert gp.get("x", 0) + gp.get("w", 0) <= 24, \
        f"Panel {p['id']} [{p['title']}] overflows grid"
```

## Grid Layout Quick Reference

```
x=0                              x=24
┌────────────────────────────────┐
│ ROW  w=24  h=1         y=N     │  ← always append row first, then y += 1
├───────┬───────┬───────┬────────┤
│ stat  │ stat  │ stat  │ stat   │  y=N+1  (8 × w=3 = 24)
│ w=3   │ w=3   │ w=3   │ w=3   │
├───────┴───────┼────────────────┤
│ timeseries    │  bargauge      │  y=N+5  (w=12 + w=12 = 24)
│ w=12, h=8     │  w=12, h=8     │
├───────────────┴────────────────┤
│  table  w=24  h=10             │  y=N+13
└────────────────────────────────┘
```

**Typical heights:** stat=3-4, timeseries=7-10, bargauge=6-10, table=8-14,
piechart=8-10, barchart=8-12, histogram=8-10, state-timeline=8-12, text=3-5.

## Panel Type Summary

| Type | Best for | Key option |
|------|----------|------------|
| `stat` | KPIs, counts, health scores | `graphMode: "area"` for sparkline |
| `timeseries` | Trends, rates, durations | `stacking: "normal"` for stacked area |
| `bargauge` | Top-N rankings, utilisation | `displayMode: "gradient"` |
| `table` | Inventory, detail, joined data | `format:"table"`, `instant:true` on targets |
| `piechart` | Distribution breakdowns | `pieType: "donut"` |
| `barchart` | Categorical counts | Needs labelsToFields+reduce transforms |
| `histogram` | Value distribution (shape) | `bucketSize: "auto"` |
| `state-timeline` | Status changes over time | `mergeValues: true` |
| `gauge` | Single value vs min/max | `showThresholdMarkers: true` |
| `heatmap` | Latency bucket heatmaps | `format: "heatmap"` on target |
| `text` | Section headers, docs | `mode: "markdown"` |
| `row` | Collapsible section divider | `collapsed: true` + child `panels:[]` |

## Transform Summary

Transforms run in sequence; each step receives the output of the previous.

| ID | Use for |
|----|---------|
| `organize` | Rename, hide, reorder columns — use on every table |
| `labelsToFields` | Pivot label values into columns (prerequisite for barchart) |
| `reduce` | Collapse time series → single value (use after labelsToFields) |
| `sortBy` | Sort rows by column descending/ascending |
| `filterByValue` | Keep/exclude rows matching conditions (AND/OR) |
| `groupBy` | Aggregate like SQL GROUP BY |
| `calculateField` | Derive new columns (ratio, %, binary math) |
| `merge` | Combine multiple query results into one table |
| `joinByField` | SQL-style JOIN on a shared field |
| `convertFieldType` | Change column type (string→number, string→time) |
| `formatString` | upperCase, lowerCase, titleCase, trim |
| `renameByRegex` | Bulk rename via regex (strip label prefixes) |
| `limit` | Keep top N rows (negative = keep from end) |
| `partitionByValues` | Split one table into N sub-tables by field |
| `labelsToFields` + `reduce` + `sortBy` | **The barchart chain** — always use together |

## Variables

```python
def var(name, label, query_str, multi=True, include_all=True):
    return {
        "name": name, "label": label, "type": "query",
        "datasource": copy.deepcopy(DS),
        "query":      {"query": query_str, "refId": "Q"},
        "definition": query_str,          # MUST match query.query exactly
        "refresh": 2,                     # 2 = on time range change
        "multi": multi,
        "includeAll": include_all,
        "allValue": ".*",                 # regex sent for "All" — required
        "current": {}, "sort": 1, "options": []
    }

# Usage in PromQL:
# Single:  {label="$variable"}
# Multi:   {label=~"$variable"}      ← always use =~ for multi-select
```

Common query strings:
```
label_values(my_info_metric, projectName)
label_values(agent_info{projectName=~"$project"}, agentName)   # cascading
label_values(builds, result)
```

## PromQL Quick Reference

**Rate then join (the golden pattern):**
```promql
rate(metric[5m]) * on(projectID) group_left(projectName) project_info{projectName=~"$project"}
```

**Dual join (add two name labels):**
```promql
metric
  * on(buildDefinitionID, projectID) group_left(buildDefinitionName) build_def_info
  * on(projectID) group_left(projectName) project_info{projectName=~"$project"}
```

**Count with filter:**
```promql
count(pullrequest_info{voteStatus="Approved"} * on(projectID) group_left(projectName) project_info{projectName=~"$project"})
```

**Age in seconds:**
```promql
time() - pullrequest_status{type="created"} * on(projectID) group_left(projectName) project_info
```

## Metric Discovery (Run Before Building)

```bash
# What metrics does this exporter expose?
curl -s "http://prometheus:9090/api/v1/label/__name__/values" | \
  python3 -c "import json,sys; [print(m) for m in json.load(sys.stdin)['data'] if 'myexporter' in m]"

# What labels does a metric actually have?
curl -s "http://prometheus:9090/api/v1/query?query=my_metric" | \
  python3 -c "
import json,sys
r = json.load(sys.stdin)['data']['result']
if r: print('Labels:', list(r[0]['metric'].keys())); print('Example:', json.dumps(r[0]['metric'], indent=2))
else: print('NO DATA')
"

# Does a summary metric have quantile labels?
curl -s "http://prometheus:9090/api/v1/query?query=my_summary" | \
  python3 -c "
import json,sys
r = json.load(sys.stdin)['data']['result']
print('quantile values:', sorted({x['metric'].get('quantile','none') for x in r}))
"
```

## Debugging Checklist

| Symptom | Cause | Fix |
|---------|-------|-----|
| Empty panel | Filter label not on metric | Join via info metric first |
| `bad_data: binary expression` | Join inside `rate()` | Move join outside range fn |
| "All" variable returns nothing | Missing `allValue: ".*"` | Add allValue to var def |
| Table shows no columns | Missing `format:"table"` + `instant:true` | Fix target options |
| Variable broken on reload | `definition` ≠ `query.query` | Make them identical |
| Collapsed row panels missing | Child panels in top-level array | Move to row `panels:[]` |
| Grid overflow | x + w > 24 | Run validation assert loop |
| Emoji causes SyntaxError | Emoji inside Python f-string | Put emoji in legend string, not PromQL |
| No quantile labels on summary | Exporter exports `_sum`/`_count` only | Use sum/count ratio |
